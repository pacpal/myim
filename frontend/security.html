1. SQL注入攻击与防御 高危

SQL注入是通过在输入中插入SQL代码来操纵数据库的攻击方式。攻击者可绕过认证、窃取或篡改数据。
攻击原理：漏洞接口使用字符串拼接构造SQL，' OR '1'='1 使WHERE条件恒真。
防御方法：使用参数化查询（Prepared Statement），输入仅作为数据而非SQL代码处理。

⚠️ 漏洞接口（字符串拼接）

SELECT * FROM users WHERE username = '" + input + "'

✅ 防御接口（参数化查询）

2. 跨站脚本攻击（XSS）与防御 高危

XSS攻击通过在网页中注入恶意脚本，当其他用户浏览时脚本被执行，可盗取Cookie或会话信息。
攻击原理：漏洞接口直接返回原始内容，前端用innerHTML渲染会执行注入的代码。
注意：浏览器安全特性规定 innerHTML 插入的 <script> 标签不会执行， 但 <img onerror>、<svg onload> 等事件型 payload 仍会触发！
防御方法：对输出进行HTML转义，将 < > 等转为实体。

⚠️ 漏洞接口（原始输出）

content = "<img src=x onerror=alert('XSS')>"

3. 拒绝服务攻击（DoS）与防御 高危

DoS攻击通过发送大量请求耗尽服务器资源，导致正常用户无法访问服务。
攻击原理：漏洞接口无任何限制，每次请求执行重计算，并发请求会耗尽CPU。
防御方法：①速率限制中间件 ②连接数限制 ③请求频率控制。

⚠️ 漏洞接口（无限制）

无速率限制，每次执行1000万次循环
等待测试...

✅ 防御接口（速率限制）

RateLimitMiddleware: 120次/分钟/IP

4. 数据篡改与哈希链检测 创新点

攻击原理：攻击者（如DBA或通过SQL注入写权限）直接修改数据库中消息的内容，但不更新哈希值。
防御原理：每条消息存储 prev_hash 和 curr_hash，形成哈希链。 curr_hash = SHA256(prev_hash || content || from_user_id || created_at)。 校验时重新计算哈希，若与存储值不符 → 内容被篡改；若 prev_hash 与上一条不符 → 链断裂（删除/插入）。
审计日志自身也受哈希链保护，防止管理员删除审计记录。

⚠️ 模拟篡改数据库

UPDATE messages SET content=? WHERE id=? (不更新curr_hash)
等待测试...

✅ 哈希链检测

recompute hash != stored curr_hash → 篡改!
等待校验...
